# Quest 01. HTML과 웹의 기초

## Introduction
* HTML은 HyperText Markup Language의 약자로, 웹 브라우저에 내용을 표시하기 위한 가장 기본적인 언어입니다. 이번 퀘스트를 통해 HTML에 관한 기초적인 사항들을 알아볼 예정입니다.

## Checklist
* HTML 표준의 역사는 어떻게 될까요?
A: HTML의 초기 버전들: HTML은 1991년에 Tim Berners-Lee가 개발한 최초의 웹 문서 언어로 시작되었습니다. 초기에는 버전 번호가 없었으며, 각각의 버전은 웹 커뮤니티의 발전에 따라 새로운 기능이나 요구 사항을 반영하여 개발되었습니다.

HTML 2.0: 1995년에 표준화되었으며, 초기 웹 페이지를 만들기 위한 기본적인 구조를 정의했습니다.

HTML 3.2: 1997년에 발표되었으며, 이미지 맵, 테이블 등과 같은 새로운 기능을 도입했습니다.

HTML 4.01: 1999년에 발표되었으며, 다양한 웹 브라우저에서 일관된 동작을 보장하기 위해 노력했습니다.

XHTML 1.0: 2000년에 발표되었으며, XML 기반의 HTML 버전으로, 엄격한 문법을 갖추고 XML의 장점을 가지고 있었습니다.

HTML5: 2014년 표준으로 발표되었으며, 오디오, 비디오, 그래픽, 드래그 앤 드롭 등과 같은 다양한 멀티미디어 기능을 지원하는 새로운 표준입니다.
___
  * HTML 표준을 지키는 것은 왜 중요할까요?
크로스 플랫폼 호환성: HTML 표준을 준수하면 모든 종류의 브라우저 및 장치에서 일관된 동작을 보장할 수 있습니다.
접근성: 표준 준수는 웹 콘텐츠의 접근성을 향상시키는 데 도움이 됩니다.
SEO: 검색 엔진은 표준 준수 웹 페이지를 더 잘 이해하고 색인화할 수 있습니다.
유지 보수 및 개발 효율성: 표준 준수는 코드의 일관성을 유지하고 개발자 간의 협업을 간소화합니다.
___
  * XHTML 2.0은 왜 세상에 나오지 못하게 되었을까요?
   A: XHTML 2.0은 웹 콘텐츠와 관련된 심각한 기술적 문제 및 논란으로 인해 완성되지 못했습니다. 이러한 문제로 인해 웹 표준화 단체는 HTML5를 개발하는 데 집중하게 되었습니다.
___
  * HTML5 표준은 어떤 과정을 통해 정해질까요?
 A: HTML5는 W3C(월드 와이드 웹 컨소시엄)와 WHATWG(웹 하이브리드 작업 그룹)라는 두 가지 주요 그룹의 협력을 통해 개발되었습니다. 이 그룹들은 웹 표준을 위한 논의와 개발을 이끌고 있습니다.
___
* 브라우저의 역사는 어떻게 될까요?
A: 최초의 브라우저들: 1990년대 초반에는 Mosaic과 Netscape Navigator와 같은 초기 브라우저들이 등장했습니다.

Internet Explorer의 독점: Microsoft의 Internet Explorer가 1990년대 후반부터 2000년대 초반까지 웹 브라우징 시장을 독점했습니다.

다양한 브라우저의 출현: Mozilla Firefox, Google Chrome, Safari, Opera 등의 다양한 브라우저들이 이후 등장하면서 시장에서의 Internet Explorer의 점유율이 감소하기 시작했습니다.

모바일 브라우저의 등장: 스마트폰과 태블릿의 보급으로 모바일 브라우저가 중요해졌고, 이에 대응하여 모바일 운영체제에 내장된 브라우저들도 많이 개발되었습니다.

현재: 현재에는 다양한 브라우저가 시장을 공유하고 있으며, 브라우저 엔진의 경쟁과 웹 표준 준수가 더욱 중요해지고 있습니다.
___
  * Internet Explorer가 브라우저 시장을 독점하면서 어떤 문제가 일어났고, 이 문제는 어떻게 해결되었을까요?

A: 웹 표준 준수 부족: Internet Explorer는 웹 표준을 충분히 준수하지 않았기 때문에 웹 개발자들은 여러 브라우저에 대응하기 위해 추가적인 작업을 해야 했습니다.
보안 취약성: Internet Explorer는 보안 취약점이 많았고, 이를 악용한 악성 코드가 전파되는 등의 보안 문제가 있었습니다.
이러한 문제는 다른 브라우저들이 출현하고, 사용자들의 선택이 분산되면서 해결되었습니다. 더 나은 웹 표준 준수와 더 나은 보안 기능을 제공하는 브라우저들이 시장에서 경쟁하게 되었습니다.
___
  * 현재 시점에 브라우저별 점유율은 어떻게 될까요? 이 브라우저별 점유율을 알아보는 것은 왜 중요할까요?

A: 순위	웹브라우저	점유율
1	크롬(구글)	63.6%
2	IE(인터넷 익스플로러)	24.6%
3	마이크로소프트 엣지	4.08%
4	사파리	3.11%
5	웨일(네이버)	1.64%  
브라우저별 점유율을 알아보는 것은 중요한 이유는 다음과 같습니다:
웹 개발 및 디자인: 어떤 브라우저를 대상으로 웹 사이트를 개발하고 디자인할지 결정하는 데 도움이 됩니다.
호환성: 어떤 브라우저가 더 많은 사용자를 보유하고 있는지 알아보면, 해당 브라우저와의 호환성을 높이는 것이 중요해집니다.
마케팅 및 전략: 웹 서비스나 앱을 제공하는 기업이 브라우저의 점유율을 파악하여 마케팅 전략을 결정하는 데 도움이 됩니다.
___
  * 브라우저 엔진(렌더링 엔진)이란 무엇일까요? 어떤 브라우저들이 어떤 엔진을 쓸까요?
A: 브라우저 엔진(렌더링 엔진)은 웹 페이지를 해석하고 표시하는 데 사용되는 소프트웨어입니다. 몇 가지 주요 브라우저 엔진은 다음과 같습니다:
Blink: Google이 개발한 엔진으로, Google Chrome과 Opera에서 사용됩니다.
WebKit: Apple이 개발한 엔진으로, Safari와 일부 다른 브라우저에서 사용됩니다.
Gecko: Mozilla가 개발한 엔진으로, Firefox에서 사용됩니다.
Trident: Microsoft가 개발한 엔진으로, 이전 버전의 Internet Explorer에서 사용되었습니다.
EdgeHTML: Microsoft Edge용으로 개발된 엔진으로, 이전 버전의 Microsoft Edge에서 사용되었습니다. 최근에는 Chromium 기반의 엔진인 Blink로 전환되었습니다.
___
  * 모바일 시대 이후, 최근에 출시된 브라우저들은 어떤 특징을 가지고 있을까요?
 A: 성능 향상: 최신 브라우저들은 더 빠르고 안정적인 성능을 제공하기 위해 다양한 기술과 기능을 도입하고 있습니다.
보안 강화: 사용자들의 개인정보와 데이터 보호를 강화하기 위해 보안 기능을 개선하고 있습니다.
모바일 최적화: 모바일 디바이스에 대한 최적화와 모바일 기기에서의 사용성을 향상시키기 위해 노력하고 있습니다.
웹 표준 준수: 최신 브라우저들은 HTML5, CSS3, JavaScript의 새로운 기능들을 더욱 잘 지원하고 있으며, 웹 표준을 준수하는 데 더욱 주력하고 있습니다.
___
  * HTML 문서는 어떤 구조로 이루어져 있나요?
    
A: <!DOCTYPE> 선언: HTML 문서의 유형을 선언합니다.
<html> 요소: 전체 HTML 문서의 시작과 끝을 감싸는 최상위 요소입니다.
<head> 요소: 문서의 메타데이터, 스타일시트 링크, 제목 등을 포함하는 부분입니다.
<body> 요소: 실제로 사용자에게 보여지는 콘텐츠를 포함하는 부분입니다.
___
  * head 에 자주 들어가는 엘리먼트들은 어떤 것이 있고, 어떤 역할을 할까요?

head 요소에는 일반적으로 다음과 같은 엘리먼트들이 자주 포함됩니다:
title: 문서의 제목을 정의합니다. 브라우저의 제목 표시줄이나 탭에 표시됩니다.
meta: 문서의 메타데이터를 정의합니다. 예를 들어 문자 인코딩, 뷰포트 설정 등을 포함할 수 있습니다.
link: 외부 스타일시트를 연결하거나, 파비콘, 스타일시트 등의 리소스를 포함할 수 있습니다.
style: 인라인 스타일시트를 정의할 수 있습니다.
___
  * 시맨틱 태그는 무엇일까요?
 
 A: 시맨틱 태그는 의미론적으로 의미 있는 요소들을 정의하는 HTML 태그를 가리킵니다. 이러한 태그들은 웹 페이지의 구조를 더 명확하게 만들어주고, 검색 엔진이나 스크린 리더와 같은 기술적 시스템이 문서를 더 잘 이해할 수 있도록 도와줍니다.
 ___
    * 시맨틱 엘리먼트를 사용하면 어떤 점이 좋을까요?
 
 SEO: 검색 엔진 최적화를 위해 시맨틱 태그를 사용하면 검색 엔진이 콘텐츠를 더 잘 이해하고 색인화할 수 있습니다.
접근성:  스크린 리더와 같은 보조 기술이 시맨틱 태그를 통해 문서를 더 잘 이해하고 사용자에게 전달할 수 있습니다.
코드 가독성: 시맨틱 태그를 사용하면 코드의 의도가 더 명확하게 드러나며, 코드를 읽고 이해하기가 더 쉬워집니다.
___
    * section과 div, header, footer, article 엘리먼트의 차이점은 무엇인가요?

A: section: 문서의 여러 섹션을 정의합니다. 주로 컨텐츠를 그룹화하거나 페이지의 특정 부분을 나타내는 데 사용됩니다.
   div: 문서의 영역을 그룹화하는 데 사용됩니다. 주로 스타일링이나 레이아웃을 위해 사용됩니다.
   header: 섹션이나 페이지의 머리말을 정의합니다. 주로 로고, 제목, 네비게이션 등을 포함합니다.
   footer: 섹션이나 페이지의 바닥글을 정의합니다. 주로 저작권 정보, 연락처 정보 등을 포함합니다.
   article: 독립적인 콘텐츠 덩어리를 나타냅니다. 주로 포럼 게시물, 기사, 블로그 글 등과 같은 콘텐츠를 포함합니다.
___
  * 블록 레벨 엘리먼트와 인라인 엘리먼트는 어떤 차이가 있을까요?
    
A: 블록 레벨 엘리먼트: 새로운 줄에서 시작되며, 부모 요소의 너비를 가득 채웁니다. 주로 구획을 나누거나 레이아웃을 만드는 데 사용됩니다. 예시로는 div, p, h1-h6, section 등이 있습니다.
인라인 엘리먼트: 새로운 줄에서 시작되지 않고, 인라인으로 텍스트와 함께 표시됩니다. 주로 텍스트 내에 다른 요소를 강조하거나 스타일을 적용하는 데 사용됩니다. 예시로는 span, a, strong, em 등이 있습니다.
